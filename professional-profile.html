Interview
Interview with Niels Hoogeveen, Data Science Consulten/Machine learning engineer.

- Bij welk bedrijf zou je willen werken?
Silicon Valley.
- Waarom daar?
Omdat Silicon Valley is een van die grootste wereldwijd centrum voor geavanceerde technologie en innovatie.
- Waar staat het bedrijf voor? (Wat doet het?)
Silicon Valley, een wereldwijd centrum van technologische innovatie, verwijst als een uitdrukking naar de industrie en de bedrijven die het thuis noemen, evenals naar een innovatieve mentaliteit, ondernemersgeest en een levensstijl die is gebaseerd op technologisch gebaseerde rijkdom.

Stel de programmeur de vraag:
- Wat vindt hij/ zij leuk aan het werk? Neils Hoogeveen Data Science Consulten/ Machine learning engineer.
Combinatie tussen wiskunde en statistisch en programmeren en wat ik leuker vind is dat dat voor een probleem, veel verschillende oplossing en is de kunst van Data Scientist om een zon effectief oplossing te vinder en die ook goed schaal baar is en gemakkelijk is om te onderhoud. De techniek is blijven stil ontwikkelen.
- Wat zijn de taken van een programmeur?
Het probleem van de klant helder te krijgen, kijken we naar wat het voor nodig is om te verhelpen bv welke data is er nodig, wat voor soort algoritmes gaan we gebruiken en het opzetten van architectuur om een oplossing in producten te brengen. Vervolgens wanneer we data van de klant krijgt dan hebben een aantal taken beginnen van de opschonen van de data (missende data opvullen) dat noem je Encoding. Features engineering (nieuwe features bouwen), features selection( het verwijderen van de features die geen toegevoegde waarden), Machine learning module bouwen, Machine learning model testen op een test data set, het performance vergelijken van verschillende machine learning models. API bouwen die gebruik kan maken van de Machine learning module en dan is het van productie

brengen in de Cloud van de Machine learning model. De laatste stap is presenteren van de resultaten aan de klant.
- Wat is nodig om een geschikte programmeur te worden?
Je moet snappen hoe het Machine modulen werken, je moet ook snelle code schrijven, unit test schrijven, goede documentatie, niet alleen een module kan bouwen maar de infrastructuur kunt bouwen, goede coding style en laatste is al het eerst zorg dat het werk en daarna ga je pas de code verbeteren en sneller maken. Werken via scrum.
